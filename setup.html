치과 관리자용

리뷰 페이지 정보 등록

치과 이름과 구글 리뷰 링크를 입력하면, 이 정보가 담긴 전용 링크와 QR 코드가 생성돼요. 이 QR 코드를 대기실이나 데스크에 두시면 됩니다.

구글맵에서 치과 이름 검색 → "리뷰 작성" 버튼 클릭 → 주소창 URL을 복사해서 붙여넣으세요.

네이버지도에서 치과 검색 → 플레이스 페이지 링크를 복사해서 붙여넣으세요. 비워두면 네이버 버튼 없이 구글만 표시돼요.

입력하시면 리뷰 페이지 상단에 홈페이지 방문 링크가 함께 표시돼요. 외국인 환자가 병원 정보를 더 찾아볼 때 유용해요. 비워두면 표시되지 않아요.

🎨 브랜딩

로고와 브랜드 컬러, 원장님 소개를 넣으면 리뷰 페이지가 우리 병원만의 톤으로 보여요.

입력하면 버튼·포인트 색상이 이 컬러로 바뀌어요. 비워두면 기본 그린 톤이 적용돼요.

현재 이 문구는 선택하신 언어와 무관하게 입력한 그대로 표시돼요. 외국인 환자 비중이 높다면 짧고 쉬운 문장을 권장드려요.

🔍 검색 노출 (SEO)

AI가 리뷰를 쓸 때 아래 지역·키워드를 자연스러울 때만 한 번 참고해요. 억지로 반복 삽입하지 않아서 리뷰가 부자연스러워지지 않아요.

📲 온라인 채널 & 리텐션

리뷰를 남긴 환자를 병원 SNS 구독자·재방문·추천으로 자연스럽게 이어줘요.

입력하면 리뷰 작성 완료 화면에 "전화로 예약하기" 버튼이 표시돼요.

⚠️ 인스타그램/블로그 링크 중 하나 이상 입력했을 때만 배너로 표시돼요. 그리고 이 이벤트는 "리뷰 작성"이 아니라 "SNS 팔로우/공유"에 대한 보상으로 문구를 구성해주세요. 구글 리뷰 정책은 리뷰 작성 자체에 대한 보상 제공을 금지하고 있어요.

🔔 관리자 알림 · 통계 (선택, 개발자 설정 필요)

환자가 "아쉬웠던 점"을 선택하면, 그 리뷰가 외부에 올라가기 전에 병원이 먼저 알 수 있도록 실시간 알림을 보낼 수 있어요. 또한 어떤 진료·스탭이 자주 칭찬받는지 익명 통계도 모을 수 있어요. (리뷰 원문이나 환자 개인정보는 전송되지 않아요)

이 기능은 보안을 위해 이 페이지가 아니라 서버(Vercel 환경변수)에서 설정해요:

1) 아래 "병원 식별자"를 하나 정해서 입력하세요 (안 정하면 병원 이름이 대신 쓰여요)
2) Vercel 프로젝트 → Environment Variables에 등록:
  • ADMIN_WEBHOOK_URL__병원식별자 = Slack/카카오워크 Webhook 주소 (부정 피드백 알림용)
  • STATS_WEBHOOK_URL__병원식별자 = 통계를 쌓을 Webhook 주소 (Google Sheets 등)
3) 병원식별자에 특수문자가 있으면 영문/숫자만 남기고 대문자로 변환해서 등록하세요 (예: bright-smile → ADMIN_WEBHOOK_URL__BRIGHT_SMILE)
